MyFlix

This was the final project that I worked on for Tealeaf Academy. This course focused on Test Driven Development (TDD) and I got to learn and utilize all kinds of new skills and technologies, such as RSpec, Capybara, Selenium, VCR, and a whole slew of testing techniques. I also got to integrate with 3rd party APIs such as MailGun, AWS S3, and Stripe for credit card processing. Lastly, I got to learn about moving beyond the typical MVC structure of smaller Rails apps and onto refactoring complex logic into objects.

Test Driven Development

I got to build this application from the ground up using TDD. I used RSpec as my testing framework. I used the following tools and techniques: Unit, functional, and integration tests, development and test databases, shoulda matchers, red / green / refactor, controller tests, object generation, refactoring with TDD, growing complexity guided by tests, macros and shared examples, feature specs, capybara, selenium, test doubles and method stubs, testing email sending, fully integrated API tests, message expectations, and mocking.

We really got to cover a lot about testing and build and app completely using TDD.

Production-Ready Process

This project allowed me to follow the process that a real software team might follow. I started out with some mocked-up HAML files and just added the functionality bit by bit, using TDD. Each new feature was a pull request against my master branch in git. I kept the master branch clean with deployable code at all times. Once I merged the new feature in, I would run the entire test suite again. If all my tests passed,Â  I then deployed the application to a staging server to make sure there were no production-time bugs I was missing. Then, once I was confident in my code I deployed to the public-facing, production server.
